Question: 1.When should accessor methods be used? 
 We should use them when:
	- Data Transport: You are creating a simple class (like a DTO) just to move 	data between a database and the screen.
- Framework Compatibility: You are using tools like Hibernate or Spring that require standard getters/setters to work.
- Read-Only Access: You want other classes to see a value but not change it   (provide a Getter only).
- Validation: You need to check if data is valid (e.g., price > 0) before saving it.
 We should avoid them when:
- It breaks Encapsulation: If you take data out, change it, and put it back, you are exposing the "guts" of your object.
- It causes "Tight Coupling": If you change a variable name, every class using its Getter will break.

2. If you create a constructor method to build a DVD by title then create a constructor method to build a DVD by category. Does JAVA allow you to do this?

No, Java does not allow this. Java will see two constructors with the exact same signature: DigitalVideoDisc(String). It will get confused and throw a compile error because it doesn't know which one to call when you pass a String.
